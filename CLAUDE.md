## Project Overview

This is a multi-stage CI/CD pipeline project that deploys a Flask web application to AWS. The stack is: Flask app → Docker → AWS (Terraform) → Ansible configuration → GitHub Actions automation.

## Architecture

**Application layer** (`app/`): Minimal Flask app with two routes (`/` shows hostname, `/health` for ALB health checks), containerized via `Dockerfile` using Python 3.9-slim.

**Infrastructure layer** (`terraform/dev/`): Terraform provisions a VPC with public/private subnets. The ALB and bastion host live in the public subnet; web servers live in the private subnet behind NAT. HTTPS is terminated at the ALB with an ACM certificate, then forwarded to port 80 on web instances. DNS is managed via Route53 (`dev.cbcnet.me`).

**Configuration layer** (`ansible/`): The playbook installs Docker on EC2 instances and runs the Flask container, mapping host port 80 to container port 5000.

**CI/CD** (`.github/workflows/cicd-pipeline.yml`): Three sequential steps on push to main:
1. Build and push Docker image to Docker Hub
2. Run `terraform apply` to provision/update infrastructure
3. Generate Ansible inventory from Terraform outputs, then run the playbook via bastion SSH proxy

## Deployment Flow

Push to `main` → GitHub Actions builds Docker image → Terraform provisions AWS infrastructure → Terraform outputs generate `ansible/inventory/dev.ini` → Ansible deploys container to web servers.

Web servers have no direct internet access; Ansible connects through the bastion host using `ProxyJump`.

## Terraform

State is stored in S3 bucket `potential-plus-tfstate-project-2026`. Key variables:
- `runner_ip`: GitHub Actions runner IP (for bastion SSH access)
- `ssh_public_key`: Key placed on EC2 instances
- `deploy_monitoring_server`: Boolean to enable optional monitoring instance
- `web_server_count`: Derived via `length()` to control instance count

The SSL certificate ARN is hardcoded in `load_balancer.tf`. AMI IDs are hardcoded for `us-east-2`.

## Required Secrets (GitHub Actions)

`DOCKERHUB_USERNAME`, `DOCKERHUB_TOKEN`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_SSH_PUBLIC_KEY`, `AWS_SSH_PRIVATE_KEY`

## Local Commands

```bash
# Run Flask app locally
cd app && pip install -r requirements.txt && python app.py

# Build Docker image
docker build -t my-homepage:latest .

# Terraform (from terraform/dev/)
terraform init
terraform plan -var="runner_ip=<ip>" -var="ssh_public_key=<key>"
terraform apply -auto-approve -var="runner_ip=<ip>" -var="ssh_public_key=<key>"
terraform destroy -auto-approve

# Ansible (inventory generated by Terraform)
ansible-playbook -i inventory/dev.ini ansible/playbook.yml \
  --user ec2-user --private-key /tmp/ssh_key.pem \
  --extra-vars "dockerhub_username=<username>"
```
